What this repo is actually doing

Raw WebGL for the render path, small doses of Three.js for camera ergonomics, and a lot of fractal abuse in fragment shaders.

01. Foundations

The repo teaches itself from first principles.

The early scenes are intentionally bare: one triangle, then a plane, then fullscreen shader work. That makes the later fractal logic feel earned instead of magic.

02. Fractal study

The Menger logic starts in 2D and stays analytic.

Pixels are classified by repeated base-3 subdivision, not by looking up baked data. The zoom scene keeps the cursor’s target centered while you dive deeper into the pattern.

03. Camera experiments

Cross-sections and cubes are camera-space prototypes.

Before the final sponge, the repo spends time on ray hits, plane slices, box faces, and camera transforms so the 3D renderer has a real foundation.

04. Final pass

The destination scene skips voids instead of brute forcing voxels.

The DDA scene treats the sponge as recursive empty space. Level-of-detail scaling and empty-block exits let the shader move fast where there is nothing to hit.

Scene ladder

Nine scenes, each one pushing the shader stack a little harder.

The GUI scene picker makes this repo feel like a lab notebook you can scrub through. It remembers your last scene, so the current target is always one click away.

01 / simple

Smoke test triangle

Just enough raw WebGL to prove the pipeline, attributes, and fragment output are wired correctly.

02 / plane

Pixel-space bridge scene

Moves toward fullscreen work with a resolution uniform and plane-sized coordinates.

03 / menger_sponge

Base-3 fractal carveout

A 2D Menger-style study that snaps resolution to powers of three so the recursion stays clean.

04 / menger_sponge_zoom

Cursor-centered zoom probe

Turns the fractal into an exploration tool instead of a static postcard.

05 / shader_cross_section

Ray-plane slice prototype

Builds camera orientation in shader space and intersects rays with bounded planes.

06 / shader_cube

Box-face hit testing

Generalizes the slice work into orthogonal cube-face intersections and face coloring.

07 / shader_cube_animated

Motion and camera orbit

Adds animated surface treatment while wheel input rotates the view around the object.

08 / shader_cube_camera

Three.js camera handoff

OrbitControls and a PerspectiveCamera feed a rotation matrix into the shader pipeline.

09 / shader_cube_menger_dda

The 3D Menger sponge payoff

A giant recursive cube, a shader-side camera, distance-aware stepping, and empty-space skipping that makes the whole repo worth opening.

Runtime behavior

Small runtime, sharp feedback loop.

Scene persistence

The selected scene is stored in localStorage, so the repo reopens where you left off instead of resetting to square one.

Global input state

Mouse position, zoom, scroll, and WASD state live in one shared runtime object and get reused across scenes.

FPS readout

The `lil-gui` panel updates an FPS field once per second, which keeps performance visible while you jump between experiments.

Use it like this

  1. Start with simple or plane if you want the repo’s foundation.
  2. Jump to menger_sponge_zoom if you care about the fractal logic before the 3D renderer.
  3. Land on shader_cube_menger_dda when you want the full payoff: orbit the camera and inspect how the sponge gets carved by empty space.
